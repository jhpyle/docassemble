# Get started

Log into the **docassemble**
[Playground](https://fordham.docassemble.org/playground){:target="_blank"}
using the email and password given to you.

The example interview, `petition-to-open.yml`, creates a [Petition To
Open] a default judgment in an eviction case.

Press the "Save and Run" button to run the interview.

On the first page, click the "template for the Petition To Open" link
to open the document template in Google Docs. This is your own version
of the document. You can edit it, and your changes will be reflected
in the document generated by the app.

Try out the app by filling in all the information provided. You can
make up junk information; just get to the end so you can see the
document that was created.

# Tips for using the Playground

You can only have one session of the Petition To Open interview
running in your web browser at one time.

Pressing <a class="btn btn-warning">Save and Run</a> will save your changes to
`petition-to-open.yml` and start a new session in a new tab; any
existing session you were running will be deleted.

If you want to test changes to your interview without restarting the
interview from the beginning, press 
<a class="btn btn-primary">Save</a>,
then switch to the tab where your interview session is running, and
reload the page.

When editing text in the Playground you can press Ctrl-f to search.

You can also search a variable name by clicking the
<i class="fa-solid fa-search"></i> icon next to the variable name
in the sidebar on the right.

You can consult the [Docassemble documentation]. There are many
code examples that you can imitate.

# Exercises

## Exercise 1: adding a disclaimer

Your boss thinks that the first page of this interview needs a
disclaimer that says that using the tool does not create an
attorney-client relationship. Use the Playground to edit the
`petition-to-open.yml` file to add this disclaimer.

## Exercise 2: adding a new variable 

Your boss tells you that a Petition To Open must not only state when
the tenant "moved into The Property," but also when the lease was
signed.

1. Edit `petition-to-open.yml` to add a "field" for the lease date.
2. Edit the Petition To Open in Google Docs to add this language to
   the *Petition of Defendant To Open Default Judgment*.

Hints:
* Variable names cannot contain spaces. Use the underscore character
  instead of a space.
* Use `move_in_date` as an example.

## Exercise 3: adding a new defense

Your boss chastises you for not including all of the possible
defenses. There is another defense that can be raised, which is that
the landlord failed to provide a "Notice to Quit" or a "Lease
Termination Notice" prior to filing the lawsuit. If a defense is not
raised in the Petition to Open, he tells you, it is waived! He hits
you with a [redwell] and you go back to your desk to add this defense.

Hints:
* Use `rental_license_defense` as an example.
* Note that defenses are raised both in the Petition and in the
  Memorandum of Law.
  * In the *Petition*, the `{% raw %}{%p if ...}{% endraw %}` and 
  `{% raw %}{%p endif %}{% endraw %}` syntax is used because raising 
  a defense adds a new "paragraph".
  * In the *Memorandum of Law*, the `{% raw %}{% if ...}{% endraw %}`
  and `{% raw %}{% endif %}{% endraw %}` syntax is used because
  raising a defense adds a new sentence.

## Exercise 4: turn on AI

Allowing the user to write their own "explanation for why you did not
appear at the court hearing" (the variable `excuse`) is not always
effective, because users do not know the legally appropriate things to
mention. AI can help with that. Change:

    code: |
      use_ai = False

to

    code: |
      use_ai = True

and re-run the interview. This means that AI will generate the
`excuse`. Toward the end of `petition-to-open.yml` you will see the
`prompt_for_ai`, which contains the instructions for the AI model
(Claude).

Consider:

1. Is there anything you would change to the `prompt_for_ai` to
   improve how the AI handles the situation?
2. Does this mean the app is no longer a "mere scrivener" and is now
   "practicing law"?

## Exercise 5: add an informational screen

When the interview asks for a signature, it does not provide context
about why. Add a screen that appears before the `signature` screen
that says something to the effect of, "on the next screen you will be
asked for your signature. Your signature needs to be placed into the
paperwork so that the court will accept it."

Hints:
* The `code` block on line 18 of `petition-to-open.yml` guides the
  order of information that the interview collects.

## Exercise 5: add a conditional warning screen

Your boss points out that the tenant will have a much easier time
opening the judgment within ten days of the judgment, because the
legal standard is more lenient. Therefore, if ten days have not
elapsed since the judgment was entered, the user should be warned to
file the Petition To Open by the deadline.

Hints:
* You will notice that date fields have `datatype: date`. Read the
  documentation for the [`date`] `datatype` for clues about how to
  perform date computations.
* The `code` block on line 18 of `petition-to-open.yml` has an example
  of conditionally showing a screen:
  
      if requested_repairs and called_licenses_inspections:
        retaliation_defense
  
  This means:
    1. First ask the `requested_repairs` question.
    2. If the answer is yes, the `called_licenses_inspections` question 
       will be asked.
    3. However, if the the answer is no, `called_licenses_inspections`
       will not be asked, because logically it doesn't matter what the
       value is.
    4. Only if both `requested_repairs` and
       `called_licenses_inspections` are both true will the
       `retaliation_defense` question will be asked.

## Exercise 6: do some Python programming

The *Rule to Show Cause* and the *Order* are fill-in-the-blank
documents that the judge will fill out. The variable `current_year` is
inserted, so that if the current year is 2026, the year 2026 will be
inserted. If the petition is filed on or after December 20, however,
it would be best to put in the next year, so that the judge doesn't
have to correct the year. Make this change to the `code` that defines
`current_year` in `petition-to-open.yml`.

Hints:
* Check out the documentation for [`today()`] and [`DADateTime`].

# Languages used

## YAML

The syntax of the `petition-to-open.yml` file is [YAML].

    question: |
      What are your favorite things?
    fields:
      - Animal: favorite_animal
      - Food: favorite_food
      - Number: favorite_number
        datatype: number

YAML resources:

* [Introduction to YAML](https://learnxinyminutes.com/yaml/)
* [Wikipedia page on YAML](https://en.wikipedia.org/wiki/YAML)

## Python

The `code` blocks in `petition-to-open.yml` are written in the
[Python] programming language.

    code: |
      # Determine which movie ratings are allowable for the user
      allowed_movies = ['G', 'PG']
      if user.age_in_years() >= 13:
         allowed_movies.append('PG-13')
      if user.age_in_years() >= 17:
         allowed_movies.append('R')
      if user.age_in_years() >= 18:
         allowed_movies.append('NC-17')

Python resources:

* [Docassemble Python documentation](https://docassemble.org/docs/code.html#python)
* [Python tutorial](https://www.w3schools.com/python/)

## Markdown

The user interface components (`question` blocks) use [Markdown]
formatting.

    question: |
      Are you *sure* you want to do **that**?
    subquestion: |
      You might want to read the
      [handbook](https://www.legalbluebook.com/).
      
      Remember the requirements are:
      
      * $10,000 in credit card debt; and
      * Residency in Florida

Markdown resources:

* [Docassemble Markdown documentation](https://docassemble.org/docs/markup.html#markdown)
* [Introduction to Markdown](https://www.markdownguide.org/basic-syntax/)

## Mako

The templating language for the user interface is [Mako].

    question: |
      Did ${ decedent } have any other pets?
    subquestion: |
      So far, you have told me about:
      
      % for pet in pets:
      * ${ pet }
      % endfor
      
      % if len(pets) > 10:
      Frankly, I think you have enough pets already.
      % endif

Mako resources:

* [Docassemble Mako documentation](https://docassemble.org/docs/markup.html#mako)
* [Mako documentation](https://docs.makotemplates.org/en/latest/syntax.html)

## Jinja2

The templating language in the Google Doc is [Jinja2], which is
commonly used for building web sites, but here it is being used in the
content of a `.docx` file.

    {% raw %}{{ respondent }} must cease{% if should_desist %} and desist{% endif %}{% endraw %}.

Jinja2 resources:

* [Docassemble Jinja2 documentation](https://docassemble.org/docs/documents.html#docx%20template%20file)
* [Jinja2 documentation](https://jinja.palletsprojects.com/en/stable/templates/)
